Project Overview


Using Chromecast, which is a wireless communication device that enables users to display media from one device to a television, implement a web based game that allows the user to use their device as a controller and their television as a display. Our main goal is to update an existing game to add chromecast support. Currently the game is played via a web browser, using keyboard controls. Our project will allow the game to be displayed on a television via a Chromecast and played with a controller displayed via chrome. Our tasks our as follows (in priority order):

  • Add Chromecast support to a pre-existing javascript game called
  • Allow user to display the game on a television screen while controlling the game remotely via the chrome web browser
  • Implement a controller displayed via Chrome web browser that communicates with the Chromecast device.
  • Update existing game to respond to a graphic controller displayed on desktop or mobile through chrome, as opposed to a keyboard
  • Create custom game, Netcast (a play on the popular ASCII graphics based game, NetHack), that can be played via Chromecast and uses pixel-based graphics via processing
    • Create a game board with several levels that increase in difficulty
    • Create a sprite that can be moved forward, backward, and jump using arrow directions displayed on the controller and that has an inventory of items
    • Create enemies that spawn randomly on the gameboard that sprite can attack by tapping button on the controller
    • Create items that are scattered across the board that can be equipped to the sprite
    • Add sprite leveling based on dungeon levels beat and amount of enemies defeated

Overview

The Chromecast is a piece of technology developed by Google that streams digital media to a television via a small dongle. This device can be controlled by a mobile phone or a computer. On its own, a Chromecast cannot do many things, but when paired with something else it turns a “dumb” TV into a “smart” TV. The goal of the project is to create a game that can be ran on the chromecast. The controller will be the touch interface of the device and the screen where the actual game is will be the TV. The current goal is to have a chromecast implementation of an existing Centipede game. This game is a recreation of the the classic arcade game Centipede. The game will undergo a thorough redesign after the chromecast implementation is done. The plan is to update the graphics, give it a dynamic window size that changes depending on the size of the screen, and add functional home and pause screens.

The main goal of this project is to learn programming with the Chromecast. The online documentation is there, the team just has to find a way to apply all of the information available to the situation at hand. None of the group members have experience developing with the chromecast, so this will be a great learning experience for everyone.

Environment

The environment that will be used for this project is the Chrome Web Browser. The applications that we will be making will be web apps hosted through github. There are many different parts that must work together harmoniously for the project to function correctly. The operating system can be anything that runs chrome smoothly. For testing, we will be using macOS Sierra and Windows 10.

There are many different languages that will be intermingling and functioning together. The following will be included in the project: js, html, css. The designs for the sprites will be made in Photoshop and Illustrator.

There will be two separate web pages that will “speak” to each other. They are called the sender and receiver. The sender is the phone or mobile device being used that sends the information and actions to the receiver so that it can handle the information and display actions. Everything for this project will be hosted through github and available on the Chrome web browser.

Module Descriptions and Data Flow


There will be two major pieces for this project, a sender and a receiver.

Sender Pseudocode
  • Display user controls
  • When screen is cast:
    • Establish connection with receiver
    • Listen for user input
    • While receiving input:
      • Package input into message
      • Send message to receiver
    • Close connection
Receiver Pseudocode
  • When screen is cast:
    • Load game
    • Establish connection with sender
    • On receive message:
      • Determine command
      • Execute command
      • Update Screen
    • Close connection
    • Display final game screen
Sender Size Estimate: 400 lines
Receiver Size Estimate: 200 lines
Game size: 2000 lines

User Scenarios

  • When user opens the link in Chrome on their phone, Chromecast immediately tries to connect.
    • If it connects, then netcast screen appears on monitor.
    • If not display error message.
  • User hits start on the phone and the monitor displays game screen (but only if chromecast is detected).
  • User hits left or right arrow on the phone, then the sprite moves left or right on the monitor.
  • User hits “A” button on the phone, sprite fires upwards on monitor
  • User hits Triangle button on phone, game pauses
  • When game is paused, must hit triangle again to restart game.
  • If the connection is dropped, the game is paused.
  • User hits rectangle button on phone, menu screen appears.
    • User hits help, displays game instructions. Must press back to exit out of screen, returns to menu screen.
    • User hits options, takes to screen showing all options available to user, user hits one to change game. Must press back to take them back to menu.
    • User hits “x” and closes menu.
  • User passes a level, must press “B” on phone to continue.
  • User wins game, must go back to title screen.
  • User loses game, must press “B” to restart game.
